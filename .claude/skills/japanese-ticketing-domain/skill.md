# japanese-ticketing-domain — 일본 라이브 티켓팅 도메인 지식

본 프로젝트의 핵심 도메인. 한국 개발자가 직관적으로 모르는 규칙들이 많아 별도 스킬로 분리한다.

## 라운드 종류 (TicketSale.type)

시간순으로 보통 다음 순서로 열린다:

| enum | 일본어 | 설명 | 보통 방식 |
|---|---|---|---|
| `FC_SENKO` | FC先行 | 팬클럽 회원 한정. 가장 먼저, 가장 좋은 좌석 가능성 | 추첨(LOTTERY) |
| `OFFICIAL_SENKO` | オフィシャル/アーティスト先行 | 비회원도 가능, 아티스트 공식 선행 | 추첨 |
| `PLAYGUIDE_SENKO` | プレイガイド先行 | e+/로손/피아 등 판매처별 선행 | 추첨 |
| `IPPAN` | 一般発売 | 일반 발매 | 선착(FIRST_COME)이 일반적, 일부 추첨 |
| `TOJITSU` | 当日券 | 공연 당일 현장 판매 | 선착 |
| `LIVEVIEWING_SENKO` | LV先行 | 라이브뷰잉 선행 | 추첨 |
| `LIVEVIEWING_IPPAN` | LV一般 | LV 일반 | 선착 |
| `STREAMING_SALE` | 配信チケット | 배포(스트리밍) 시청권 | 선착 |

## 방식 (TicketSale.method)

- **LOTTERY** (추첨/抽選): 신청 → 마감 → 발표 → 입금 → 발권. 사용자에게 중요한 시각: **마감(ends_at)**과 **발표(announce_at)**.
- **FIRST_COME** (선착/先着): 시작 시각에 클릭 전쟁. 중요 시각: **시작(starts_at)**.

UI는 방식별로 다른 표시를 한다:
- 추첨: "마감 D-3", "발표 4/22"
- 선착: "발매 D-3 10:00 (JST)"

## 시각 컨벤션

- DB는 UTC 저장, 표시는 JST 고정. 한국 시각으로 보여주면 안 된다 (티켓팅 마감 혼동 위험)
- 추첨 라운드의 시간 4종을 모두 다룸:
  - `starts_at`: 신청 시작
  - `ends_at`: 신청 마감
  - `announce_at`: 당첨 발표
  - `payment_deadline_at`: 입금 기한
- 입금 안 하면 자동 캔슬되고 "敗者復活"(패자부활) 라운드가 별도 열리는 경우도 있음 → 새 라운드로 별도 입력

## 매수 제한 / 본인 확인 (자유 텍스트)

`Live.ticket_restrictions`에 자유 텍스트로 저장. 흔한 케이스:

- "1人4枚まで" (1인 4매)
- "本人確認あり (顔写真付き身分証必須)" (본인 확인 — 사진 신분증 필수)
- "同行者もFC会員必須" (동행자도 FC 회원 필수)
- "発券後の譲渡不可" (발권 후 양도 불가)
- "中学生以上有料、未就学児膝上鑑賞可" (중학생 이상 유료, 미취학 무릎관람 가능)

## 라이브뷰잉 / 배포 (LiveFormat)

- 실황(LIVE_VENUE)이 있는 큰 공연은 **거의 항상** 라이브뷰잉(LIVE_VIEWING)이 같이 열림
- 전국 5~30관 정도. 본 모델에선 LiveFormat 1개로 "전국 N관" 묶어 표현하고, 개별 극장 리스트는 url로 외부 안내
- 배포(STREAMING)는 ABEMA, ニコニコ, 公式 등. 아카이브 시청 기간이 있어 마감일이 라이브 종료 직후가 아닐 수도

## 알아둘 엣지케이스

1. **추첨 후 일반 발매 안 열림**: 인기 공연은 FC선행+추첨선행만으로 매진, 일반발매 없음. 이 경우 IPPAN 라운드 자체가 없음.
2. **同行者登録**: FC선행/OFFICIAL_SENKO에서 동행자를 신청 시점에 같이 등록해야 함. UI에서는 별도 안내.
3. **复活 라운드 (敗者復活/抽選漏れ救済)**: 입금 안 한 사람 좌석 회수 후 재추첨. 별개 TicketSale로 입력.
4. **先着 라운드의 整理番号**: 선착이라도 입장은 번호 순. SUMMARY에 "整理番号制"로 안내.
5. **オールスタンディング**: 좌석 없이 전부 스탠딩. tier는 1개로 합쳐도 됨.
6. **2公演制 (1day 2公演)**: 같은 날 2회 공연 (例: 昼/夜). 별개 Live로 입력.

## 한국어 표시 컨벤션

UI 표시:
- "선행 추첨" / "일반 발매" / "당일권" (한자 표기 X)
- "추첨" / "선착" 으로 방식 명확히 표시
- 일본어 원문은 툴팁이나 부가 표시로
- 가격은 ¥로 (KRW 환산 X)

## 입력 흐름 가이드 (어드민)

새 Live 등록 시 권장 순서:
1. 헤더(제목/날짜/장소)
2. 출연 밴드
3. 포맷 (실황 자동 생성됨, LV/배포 필요 시 추가)
4. 티어 (포맷별로)
5. 라운드 (FC선행 → 복제 → 일반발매 → ...)
6. 매수제한 텍스트
7. 공개
